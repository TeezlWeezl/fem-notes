this Keyword

00:06:08

The this binding can be tricky when encapsulating functions. In the old days there were often used patterns as below

myObj5 = {
    counter: 0,
    __proto__: {
        increment: function() {
            const that = this
            function increment() {
                that.counter++
            }
            increment()
        }
    }
}

this is because the this binding refers to the surrounding object this => Function => Object

One way would be to call increment.call(this) or define the function as an arrow function instead

00:10:55

Difference between __proto__ and prototype:

  • both are related to the JS protoype chain, but have different purposes

  • The prototype property is used in constructor functions.

  • Every function in JavaScript has a prototype property, and when you create an object using the new keyword with that function, the newly created object inherits properties and methods from the constructor’s prototype.

  • The __proto__ property is used to access and manipulate the prototype chain directly on an instance of an object.
