Working with Promises

00:03:39

FULFILLED: This state represents that the asynchronous operation completed successfully. When a promise is fulfilled, it means that the value promised is available and can be accessed through the then callback

RESOLVED: The term “resolved” is more general and can be used to describe a promise that has either been fulfilled or rejected. In other words, a promise is considered resolved when it reaches either the “fulfilled” or “rejected” state.
